This is to instruct how to use the API "Library".

I. Data base creation.
Before start the application the database for books and users have to be initialised. It can be obtained by runing the db-init.py program located in utils modul. This feature can be run only once before running the main application. The data will be stored in users and books tables in the dbase.db database.

II. Login and access to the modification options.
When users log in to the application, their authenticated state has to be recorded in the user session, so that it is remembered as they navigate through different pages. Flask-Login is a small but extremely useful extension that specializes in managing this particular aspect of a user authentication system, without being tied to a specific authentication mechanism.

III. Credentials reset.
Once user log in to the application, he/she can change their passwords periodically. This is an easy feature to implement, because as long as the user is logged in, it is safe to present a form that asks for the old password and a new password to replace it. As part of this change, the “Logout” link in the navigation bar is refactored into a dropdown that contains the “Change Password” and “Logout” links.

IIII. Password forgotten.
To avoid locking users out of the application when they forget their passwords, a password reset option is offered. When a user requests a password reset, an email with a reset token is sent to the registered email address. This is the Admin's job to send back new password to the user after the token is verified.
